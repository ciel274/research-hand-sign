STEP 1: LOCAL RECORDING

手話データ収録と可視化

Webカメラの前に手をかざし、手話ジェスチャーを3Dデータとして記録します。ここではAIの学習自体は行わず、手本のデータ（教師データ）を生成することを目指します。

ダッシュボードサーバー
FastAPIで構築されたローカルWebサーバー。ユーザーにUIを提供し、カメラ映像から手指の骨格座標をリアルタイムキャプチャします。
src/ui/dashboard.py
生モーションデータ
手の関節21箇所のX, Y, Zの3D座標データ（合計63次元）が、時系列のCSVファイルとしてフォルダ内に蓄積されます。
data/raw/
STEP 2: PREPROCESSING

幾何正規化とポーズトークン化

収録した生データは、カメラとの距離や手の位置、大きさによって数値がバラバラです。これらを数学的に整理し、VLA（大規模言語モデル内蔵）が読み込めるようにテキスト情報へ「翻訳」します。

幾何正規化 (前処理)
手首を原点とし、中指の長さで全体のスケール（縮尺）を統一。これにより「カメラとの距離や位置のズレ」が100%相殺され、学習が非常に安定します。
data/preprocessed/
VLA用データセット変換
時系列データを、K-meansで代表ポーズのトークンID（例：<pose_23>）に離散化し、「ひらがなの『あ』を手話で表現 ➔ <pose_23>...」というVLA専用の対話型JSONLデータを作成します。
src/learning/vla_dataset.py
STEP 3: CLOUD SYNC

データとコードのGitHub同期

ローカルPCで収集・変換したデータセットと最新のプログラムを、強力なGPUを積んだGoogle Colab（クラウド環境）へ受け渡すためのブリッジを行います。

Git / GitHub
ローカルPC側で作成した最新の VLA用データセット（jsonlファイル）や、アップデートされたプログラムコードをGitHubリポジトリにプッシュします。
requirements.txt
Google ColabのOS環境が壊れないよう、NumPy 1.x系やTransformers 4.x系などの互換性のあるライブラリ定義を同期し、Colab側で再現します。
requirements.txt
STEP 4: VLA TRAINING

Google ColabでのVLA微調整と評価

本番の学習ステップです。70億パラメータを持つ巨大なVLAモデル（OpenVLA）に手話を教え込みます。LoRAという手法を用いて、LLMアテンション層の一部のみを微調整することで、少量データでも高速に動作獲得を実現します。

VLA LoRA 実行ノートブック
Colab上のメイン制御シート。GitHubからデータを取り込み、LoRAのランク変更（実験B）や指示文セマンティクス（実験C）の検証ループを回し、評価ロスグラフ（Train vs Val）を出力します。
src/learning/VLA_LoRA_Notebook.ipynb
Google Drive同期マウント
学習が終わったモデルの追加パラメータ（LoRAの重み）を自動でGoogle Drive（マイドライブ直下の共有フォルダ）に転送・永続化保存します。

🔑 本研究の本質：マルチモーダルVLAの仕組み

本研究では、VLA（Vision-Language-Action）モデルを「テキスト指示から手の3Dモーションを直接生成するアクションジェネレータ（L ➔ A）」として動作させ、その学習メカニズムを検証します。

L (Language)

指示文テキスト入力

「『あ』を表現して」という言語文脈からLLMの事前知識を呼び出します。

🧠 OpenVLA (LLM)

LからAへの変換デコーダ

言語指示に基づき、どの関節をどう動かすべきかを「文脈予測」として決定します。

A (Action)

動作トークン出力

<pose_23> などの代表ポーズの時系列データとして、ロボット制御用の動作信号を出力します。

🗣️ 【今後の展望・発展】音声認識（Speech-to-Text）とのシームレスな統合

将来的に「音声」で手話ロボットを動かしたい場合は、音声波形を直接入力するのではなく、高性能音声認識モデル（Whisper等）をVLAのフロントエンドに結合します。 音声入力を「テキスト指示文（L）」に変換してからVLAへ流し込むことで、モデル自体の追加学習なしで「音声で完全にコントロールできる手話生成AIシステム」へ容易に拡張可能です。
