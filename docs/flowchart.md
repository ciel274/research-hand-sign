## 💡 2. そもそも「LoRAファインチューニング」とは？

本研究でモデルを賢くするために使っている **LoRA（Low-Rank Adaptation）ファインチューニング** についての解説です。

* **ファインチューニング（追加学習）とは**:
  * すでにインターネット上の膨大なデータで賢くなっている巨大AIモデル（OpenVLA-7B）に対して、新しく手話のような「特定の専門動作」を追加で覚え込ませる作業です。
* **従来のやり方の課題（全パラメータ更新）**:
  * 70億個もあるAIの脳細胞（パラメータ）をすべて書き換えるため、超高性能なスーパーコンピュータが必要になり、お金と時間がかかりすぎて学生の研究室では実行できません。
* **LoRA（低ランク適応）の仕組み（SDカードの挿入）**:
  * 元の巨大な脳（70億個）は**一切書き換えず（ロック）**、その横に**「手話動作専用の薄くて小さな追加の脳（アダプター）」**を取り付けます。
  * **たとえ話**: スマホ本体のOSを丸ごと作り直すのではなく、**「手話アプリを追加でインストールする」**ようなイメージです。
  * **メリット**: 調整するパラメータ数が1000分の1以下に激減するため、Google Colabの無料GPUを使っても、わずか**約4分**という圧倒的なスピードで学習が完了します。

## 🔬 3. 数値を書き換えて行う「3つの比較実験」の詳細

卒論の核心となる3つの実験について、何を変えて何を比較するのかを噛み砕いて説明します。

### 🧪 【実験A】手の動きの「細かさ」の比較（クラスタ数 K）
手の形の時系列データを、何種類の「基本ポーズトークン（手の形の単語）」に分類してテキスト化するかを比較します。
* **書き換える場所**: ローカルPCの `src/learning/vla_dataset.py` 内の `n_clusters`
* **変更する数値**: **`32`**（粗い） vs **`64`**（普通） vs **`128`**（細かい）
* **何を比較し、どう考察するか**:
  * 数値が小さすぎると、手の形がざっくりしすぎて手話が表現できません。逆に大きすぎると、AIが覚える単語（手の型）が多すぎて学習が難しくなります。
  * 「手の動きの再現度」と「AI의 学習しやすさ」の**ベストなバランス（黄金比）**を特定します。

### 🧪 【実験B】追加する「脳の記憶容量」の比較（LoRA ランク r）
新しく手話を覚えるために追加する「ミニ脳（LoRAアダプター）」の記憶容量（表現力）の大きさを変更して比較します。
* **書き換える場所**: Colabノートブックの `Experiment_Type = "compare_lora_rank"` 実行時、または `get_lora_config(rank)` の `r`
* **変更する数値**: **`8`**（省容量） vs **`16`**（普通） vs **`32`**（大容量）
* **何を比較し、どう考察するか**:
  * 記憶容量 `r` を増やすと複雑なジェスチャーを細かく表現できますが、過学習（丸暗記）しやすくなります。
  * 少量データで最も効率的（Val Lossが最も低くなる）な**最適な記憶容量（ランク値）**を突き止めます。

### 🧪 【実験C】「言葉の意味の理解度」の比較（プロンプトの違い）
AIに手話を指示する際、「無機質な記号」を渡すのと、「意味のある日本語の文章」を渡すのとで、学習効率に差が出るかを比較します。
* **書き換える場所**: Colabノートブックの `Experiment_Type = "compare_prompt_style"` を実行
* **変更する設定**: 
  * **`Simple ID`**: 意味を持たないただの記号（例：`Instruction: ClassID_01`）
  * **`Natural Japanese`**: 意味のある指示文（例：`Instruction: ひらがなの「あ」を手話で表現してください`）
* **何を比較し、どう考察するか**:
  * VLAは、元々巨大な「言語モデル（LLM）」を内蔵しています。そのため、意味のある日本語を渡した方が、AIが「あ」という言葉の意味をヒントにして、手の動きを早く正確に覚えられます。
  * **「言語の理解（L）が、物理アクション（A）の獲得を助けていること」**をグラフの収束速度の差で証明します。

## 🧠 4. Google Colabで VLAモデルの「どこを」「どのように」書き換えているのか？

この研究において、Google Colab上のプログラム（LoRA）がVLAモデルの内部をどう弄っているのかの詳細解説です。

### ① モデルの「どこを」書き換えているのか？
OpenVLA-7Bは、画像を見る「視覚エンコーダ（SigLIP：約27億パラメータ）」と、言葉を処理する「言語モデル（Llama-2：約70億パラメータ）」が合体したモデルです。
私たちは、**視覚エンコーダ部分には一切手を触れず（フリーズ）、言語モデル（LLM）の内側にある「自己アテンション層（Self-Attention Layers）」のみを書き換え対象**に指定しています。

具体的には、プログラム中の以下の定義がそれを示しています。
```python
target_modules = ["q_proj", "v_proj", "k_proj", "o_proj"]
```
これはLLMの中で「どの単語とどの単語が強く関連しているか（文脈）」を計算している、最も知的な神経ネットワーク層をピンポイントで指しています。

### ② モデルの「どのように」書き換えているのか？
70億個の神経ネットワークを直接書き換えるのではなく、アテンション層の横に**「手話の動き（アクション値）を出力するようにバイパス（迂回）させる追加回路（LoRA行列）」**を設置します。

1. コラボで `"ひらがなの『あ』を手話で表現してください"` というテキストを入力します。
2. 本体の言語モデルは、通常ならテキストで返答をしようとします。
3. しかし、アテンション層に取り付けられたLoRAの追加回路がアテンションの数値を引っ張り、答えをテキストではなく、**手の3D関節に割り当てられた特別な動作トークン（例：`<pose_23> <pose_5>`）の並びとして出力するように誘導**します。

この「言語の入力文脈」を「身体の動作出力」へマッピングし直す接続回路をColab上で動的に構築し、学習させるのが今回の研究の技術的実態です。
